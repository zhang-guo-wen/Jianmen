# Security Policy

## Supported Versions

Jianmen 尚未发布正式版本，当前仅有开发分支。

| Version | Supported          |
| ------- | ------------------ |
| dev     | :white_check_mark: |

## Reporting a Vulnerability

请勿公开提交安全漏洞 Issue。

请将漏洞报告发送至项目维护者。我们会在 48 小时内确认收到，并在 7 天内给出初步评估和修复计划。

### 漏洞报告应包括

1. 受影响的版本/commit
2. 漏洞描述和复现步骤
3. 潜在影响范围
4. 建议的修复方案（如有）

### 安全设计原则

- API 不泄露密码、私钥内容、私钥解锁口令
- 审计日志为 append-only，会话结束后计算 hash chain
- 默认不录制客户端按键输入，避免密码写入录像
- 私钥路径仅作为历史兼容字段，不在产品入口暴露

## 已知安全边界

- 本项目处于早期开发阶段，尚未经过完整安全审计
- 生产环境部署前请完成独立安全评估
- 数据库代理 TLS 场景下暂不支持 SQL 审计（流量透传）
