# 批量新建用户 — 设计规格

日期：2026-07-09

## 概述

在用户管理页面增加"批量新建"功能，用户输入多行中文名字，自动转换为拼音账户名并生成随机密码，以可编辑表格形式预览后逐条创建。

## 交互流程

```
[批量新建] 按钮 → 弹窗打开
  ├── 多行文本输入框（输入中文/英文名字，每行一个）
  ├── 点击 [格式化]
  ├── 文本替换为可编辑表格
  │   ├── 名称（display_name）
  │   ├── 账户名称（username，拼音/字母，无空格）
  │   ├── 随机密码（password，12位）
  │   └── 操作（删除行）
  └── 点击 [保存] → 逐条 POST /api/users
      └── 每行显示创建状态（成功 ✓ / 失败 ✗ + 错误信息）
```

## 拼音转换规则

- 纯中文 → 拼音（pinyin-pro 库），去空格（如"张三" → `zhangsan`）
- 纯字母 → 保留原样（如"john" → `john`）
- 中英混合 → 中文部分转拼音，英文保留（如"李David" → `lidavid`）

## 随机密码规则

- 12 位，包含大小写字母、数字、特殊符号
- 前端 `crypto.getRandomValues()` 生成

## 表格行为

| 特性 | 说明 |
|------|------|
| 编辑 | 所有单元格可编辑（名称、账户名称、密码） |
| 删除行 | 每行有删除按钮，可移除不需要的行 |
| 重复检测 | 格式化时自动跳过完全重复的中文名 |
| 去空格 | 账户名称自动去除首尾空格 |

## 保存逻辑

- 前端逐条调用 `POST /api/users`，请求体 `{ username, password, display_name }`
- 每行独立状态：未创建 / 创建中 / 成功 / 失败
- 失败行显示后端返回的错误信息
- 已成功的行不可重复创建
- 全部完成后刷新用户列表

## 依赖

## 后端

- **无需改动**，复用现有 `POST /api/users` 接口
