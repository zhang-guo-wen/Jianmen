# 个人设置页客户端激活流程改版设计

## 背景

当前个人设置页将全局“保存配置”按钮放在卡片头部，并在 SSH 客户端、数据库客户端页签中内嵌展示协议注册命令、复制按钮和“我已执行命令，保存到浏览器”按钮。这个交互存在几个问题：

1. 保存入口是全局按钮，但客户端配置实际是分页签独立操作，用户不容易理解当前保存的是哪一部分。
2. 协议注册命令长期占据页面正文，视觉负担大，且和“保存配置”的主流程割裂。
3. 客户端相关配置需要先执行本地协议注册，再把对应配置保存到后端并写入当前浏览器，但现有流程把“保存”和“激活确认”拆成了两个位置。
4. 当前 SSH 客户端选项、平台选项和数据库客户端字段文案与新的产品要求不一致。

本次改版聚焦个人设置页中的三个页签：界面与终端、SSH 客户端、数据库客户端。目标是在不调整后端接口结构的前提下，重构前端交互，使客户端配置的保存与激活确认在当前页签内闭环完成。

## 目标

### 必须实现

1. 去掉卡片头部的全局“保存配置”按钮。
2. 在每个页签内容区顶部右侧提供独立保存按钮。
3. “界面与终端”页签点击保存时直接保存。
4. “SSH 客户端”和“数据库客户端”页签点击保存时先弹出激活弹窗。
5. 只有在弹窗内点击“已激活”后，才真正把当前页签相关字段保存到后端并写入当前浏览器。
6. 协议注册命令、复制命令按钮、激活确认按钮全部从页签正文迁移到弹窗内。
7. SSH 客户端默认使用 Xshell，下拉中不再提供“系统默认 SSH 协议”“系统 SSH (ssh.exe)”两个新选项。
8. SSH 客户端和数据库客户端的 macOS、Linux 平台选项都改成禁用。
9. 数据库客户端路径输入框增加“选择文件”按钮，并统一相关帮助文案。
10. 保留旧配置兼容行为，不自动改写历史值。

### 明确不做

1. 不修改后端 `UserPreferences` 接口结构。
2. 不新增新的系统设置页或子路由。
3. 不对客户端协议命令生成逻辑做协议级重写，只调整触发时机和展示位置。
4. 不清理历史旧值，不做自动迁移脚本。
5. 不为这次需求引入额外的跨页面状态管理抽象。

## 相关现状

- 页面入口：`web/src/views/SettingsView.vue`
- 用户偏好存储：`web/src/stores/preferences.ts`
- 数据库客户端本地注册状态：`web/src/stores/databaseClient.ts`
- SSH 客户端选项与注册命令：`web/src/config/sshClients.ts`
- 数据库客户端选项与注册命令：`web/src/config/databaseClients.ts`

当前页面中，`SettingsView.vue` 既承担表单编排，也承担注册命令展示与确认逻辑。`ClientRegistrationAlert` 是页内内嵌组件，不是对话框。`saveAll()` 会把整页表单一次性提交；`confirmSSHRegistration()` / `confirmDBRegistration()` 会在用户确认执行命令后调用 `preferences.loadToBrowser()`，把数据库中的完整配置写入浏览器缓存。

## 设计概览

### 总体思路

保留 `SettingsView.vue` 作为个人设置路由页，但将保存行为按页签拆开。页面仍然统一维护表单状态和字段校验，激活命令的生成逻辑继续使用现有配置模块；变化点在于：

1. 顶部全局保存按钮改为每个页签各自的顶部保存按钮。
2. 客户端相关命令区从正文移除，改成弹窗式激活流程。
3. 客户端页签的真正保存动作后移到“已激活”按钮点击时执行。
4. 提交到后端和浏览器缓存的字段范围严格收敛到当前页签。

### 推荐实现方式

采用轻量重构：

- `SettingsView.vue` 继续持有整体表单、校验和页签状态。
- 新增一个通用弹窗组件 `ClientActivationDialog.vue`，只负责显示命令和激活按钮，不承担业务提交。
- 现有 `ClientSectionHeading` 扩展为支持在右侧插槽中放状态标签和保存按钮。
- `preferences` store 保留现有 `fetch` / `update` / `loadToBrowser`，补充只向浏览器写入部分字段的能力，避免客户端页签确认时把整份配置无差别覆盖到浏览器。

这样可以满足需求，又避免把当前页面拆成过多 composable 或子组件。

## 组件与职责

### `SettingsView.vue`

继续作为页面编排层，负责：

1. 管理 `activeTab`、表单副本、字段校验状态。
2. 根据当前表单计算 SSH / 数据库注册命令。
3. 控制两个客户端激活弹窗的显隐状态。
4. 区分三个保存入口：
   - 保存界面与终端
   - 打开 SSH 激活弹窗
   - 打开数据库激活弹窗
5. 在弹窗确认时仅提交当前页签字段。
6. 根据保存结果更新本地注册状态与浏览器缓存。

### `ClientSectionHeading`

保留为轻量展示组件，但结构需要调整：

- 左侧仍显示标题和说明。
- 右侧默认显示状态标签。
- 通过插槽或可扩展结构允许在右侧追加保存按钮。

该组件不负责点击后的业务行为，只作为统一的标题栏布局容器。

### `ClientActivationDialog.vue`

新增通用对话框组件，负责：

- 显示标题。
- 显示固定提示文案：`请执行协议注册命令，激活本地客户端`。
- 展示只读命令文本框。
- 提供“复制命令”“已激活”两个按钮。
- 向父组件抛出 `copy` 与 `confirm` 事件。

该组件不直接访问 store，也不感知 SSH / 数据库的差异，差异由父组件通过 props 注入。

## 页签级保存与激活流程

### 界面与终端页签

字段范围：

- `theme`
- `terminal_font_family`
- `terminal_font_size`

流程：

1. 用户点击标题右侧“保存配置”。
2. 前端只提交上述三个字段。
3. 保存成功后更新 store 和界面主题、终端字体。
4. 不弹窗，不改动 SSH / 数据库协议注册状态。

即使其他页签存在未提交改动，该页签保存也只影响本页字段。

### SSH 客户端页签

字段范围：

- `ssh_client`
- `ssh_client_platform`
- `ssh_client_path`

流程：

1. 用户点击标题右侧“保存配置”。
2. 前端先校验 SSH 相关字段；有错误则直接提示，不弹窗。
3. 校验通过后，打开 SSH 激活弹窗。
4. 即使当前表单值与后端已保存值相同，也照样弹窗。
5. 用户在弹窗内点击“复制命令”时，仅复制当前计算出的 SSH 协议注册命令。
6. 用户点击“已激活”后：
   - 再次校验字段；
   - 只提交 SSH 相关字段到后端；
   - 保存成功后只把 SSH 相关字段写入当前浏览器缓存；
   - 将 `sshProtocolRegistered` 标记为 `true`；
   - 关闭弹窗并提示成功。

如果用户修改过 SSH 客户端、平台或路径，则在表单阶段仍然保持未激活状态；只有确认成功后才写入激活标记。

### 数据库客户端页签

字段范围：

- `db_client`
- `db_client_platform`
- `db_client_path`
- `db_client_ca_file_path`

流程：

1. 用户点击标题右侧“保存配置”。
2. 前端先校验数据库客户端相关字段；有错误则直接提示，不弹窗。
3. 校验通过后，打开数据库激活弹窗。
4. 即使当前表单值与后端已保存值相同，也照样弹窗。
5. 用户点击“复制命令”时，仅复制当前数据库协议注册命令。
6. 用户点击“已激活”后：
   - 再次校验字段；
   - 只提交数据库相关字段到后端；
   - 保存成功后只把数据库相关字段写入当前浏览器缓存；
   - 将数据库协议注册状态标记为 `true`；
   - 关闭弹窗并提示成功。

数据库客户端页签的命令弹窗不会再在正文中长期占位。

## 数据流与 store 调整

### 后端保存策略

继续复用 `preferences.update(patch)`，但每次只传当前页签需要保存的字段：

- 界面与终端：传 3 个字段。
- SSH 客户端：传 3 个字段。
- 数据库客户端：传 4 个字段。

这样既不需要修改后端接口，也符合“只保存当前页签相关字段”的要求。

### 浏览器缓存策略

当前 `preferences.loadToBrowser()` 会重新拉取完整偏好并把完整配置写入浏览器缓存，不适合新的页签级确认流程。这里需要拆分职责：

1. 保留 `loadToBrowser()` 给需要完整同步的旧流程或其他使用方。
2. 在 `preferences` store 中新增一个内部统一的字段级缓存写入方法：`persistPartialToBrowser(patch)`。
3. 页面层在不同页签确认成功后，只传入当前页签字段给 `persistPartialToBrowser(patch)`。

要求：

- 只覆盖当前页签相关字段。
- 不覆盖其他页签尚未提交的临时表单值。
- 浏览器缓存里的主题和终端设置逻辑保持不变。

### 协议注册状态策略

- SSH：继续使用 `preferences.markSSHProtocolRegistered(true/false)`。
- 数据库：继续使用 `databaseClient.markRegistered()` / `markUnregistered()`。

字段发生变化时，应继续清空对应激活状态，保持“配置变化后必须重新激活”的规则。

## 兼容性规则

### SSH 客户端旧值兼容

新的 SSH 客户端下拉中移除以下两个选项：

- `系统默认 SSH 协议`
- `系统 SSH (ssh.exe)`

兼容规则：

1. 如果后端返回旧值 `default` 或 `system`，页面加载时保留该值显示。
2. 不自动替换成 `xshell`。
3. 新下拉列表中不再允许新选中这两个值。
4. 一旦用户改选其他客户端，就不能再选回这两个旧值。

为了保证表单在旧值场景下仍可显示，需要在下拉选项渲染时动态补一个“当前旧值只读项”，或通过 `computed` 构造“可选项 + 当前历史值”列表，而不是直接使用静态数组。

### SSH 默认值规则

对于没有保存过 SSH 客户端的新用户，默认值应为 `xshell`。这意味着：

- 前端默认值层与页面表单初始化层都要把空值补成 `xshell`。
- 后端如果返回空字符串，页面装载后也应在表单副本中显示为 `xshell`。
- 不能影响旧用户已经保存的历史值 `default`、`system` 或其他客户端值。

默认值补齐应发生在前端表单层，而不是通过自动回写后端完成。只有当用户在 SSH 页签弹窗中点击“已激活”后，当前表单值才会真正提交到后端。

### 平台旧值兼容

SSH 与数据库客户端两个页签都只支持 Windows 激活流程，因此平台选择需要变更为：

- `Windows` 可选。
- `macOS`、`Linux` 在选项中显示但禁用。

兼容规则：

1. 如果当前保存值是 `macos` 或 `linux`，页面仍显示该值。
2. 不自动改成 `windows`。
3. 用户重新选择后，只能改成 `windows`。
4. 在旧平台值场景下，不生成协议注册命令；用户需要先改到 `windows` 才能继续激活保存。

## 校验规则

### SSH 客户端

沿用现有路径校验逻辑，但需要补充一条客户端类型约束：

- 路径必须是 Windows `.exe` 绝对路径。
- 平台不是 `windows` 时不能生成命令。
- 当前客户端如果是历史值 `default` / `system`，页面允许显示该值，但不允许继续走激活保存流程。

注意事项：

- 当历史值是 `default` / `system` 时，点击保存配置应提示用户先改选受支持的 SSH 客户端，而不是弹出空命令弹窗。
- 只有当前客户端是受支持的本地客户端且路径、平台都合法时，才允许弹出激活弹窗。

### 数据库客户端

沿用现有路径与 CA 文件路径校验逻辑：

- 客户端路径必须是合法绝对路径。
- CA 文件路径在非空时必须是合法证书文件绝对路径。
- 平台不是 `windows` 时不能进入激活保存流程。

数据库客户端路径增加“选择文件”按钮后，仍然允许用户手动粘贴覆盖。

## 风险与处理

### 风险 1：旧值显示与选项列表冲突

如果直接删除静态选项，Element Plus 的下拉在当前值不在选项列表时可能出现显示异常。实现时必须保证“当前历史值”仍能作为显示项存在，但不可再次被普通用户新选中。

处理方式：使用计算属性构造展示列表，在检测到当前值是历史值时临时追加只读选项。

### 风险 2：部分字段写入浏览器缓存导致其他字段丢失

如果仍沿用整对象覆盖式写缓存，会把其他未确认字段或旧缓存误清掉。

处理方式：浏览器缓存写入必须使用字段级合并，而不是整对象重置。

### 风险 3：无修改也弹窗容易被误判为多余操作

这是明确需求，因此实现时不能加“无变更不处理”的短路逻辑。

处理方式：客户端页签保存入口始终按“校验成功即弹窗”处理，是否有差异只影响协议状态重置，不影响弹窗出现。

## 验收标准

满足以下条件即可认为本次设计完成：

1. 卡片头部全局“保存配置”按钮消失。
2. 三个页签各自拥有顶部保存按钮。
3. 界面与终端保存为直接提交。
4. SSH 与数据库客户端保存为“先弹窗，后激活确认，最后提交”。
5. 弹窗中承载协议注册命令、复制按钮和“已激活”按钮。
6. 点击“已激活”时只保存当前页签字段到后端和浏览器。
7. SSH 默认 Xshell，且不再提供两个系统级 SSH 新选项。
8. SSH 与数据库平台中的 macOS、Linux 都为禁用。
9. 数据库客户端路径支持“选择文件”。
10. 旧值场景可以展示，不自动迁移。
