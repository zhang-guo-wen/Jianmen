## 一、连接账号编号规范

### 1.1 格式定义

```
<资源前缀><资源ID><会话ID>
 1位字母   4位     5位
```

- **总长度**：10 位字符
- **字符集**：`0-9 a-z A-Z`（62 个字符），区分大小写，无特殊字符
- **完整用户名示例**：`HaB12cD345`、`D000200009`

### 1.2 资源前缀

| 前缀 | 资源类型 | 说明 |
|---|---|---|
| `H` | host_account | 主机账号 |
| `D` | database_account | 数据库账号 |

### 1.3 编码示例

| 场景 | 前缀 | 资源ID | 会话ID | 完整用户名 |
|---|---|---|---|---|
| 主机永久账号 | H | aB12 | cD345 | `HaB12cD345` |
| 主机临时授权 | H | aB12 | xY987 | `HaB12xY987` |
| 数据库永久账号 | D | 0002 | 00001 | `D000200001` |
| 数据库临时授权 | D | 0002 | 00009 | `D000200009` |

## 二、资源ID 生成规则

### 2.1 生成方式

按资源类型独立自增，转为 4 位 62 进制字符串（不足左补 `0`）。

| 资源类型 | 前缀 | 取值范围 |
|---|---|---|
| 主机账号 | H | `0001` ~ `zzzz`（62⁴ ≈ 1476 万） |
| 数据库账号 | D | `0001` ~ `zzzz`（62⁴ ≈ 1476 万） |

### 2.2 生成逻辑

1. 创建主机账号时，查找该类型当前最大自增序号，自增得到下一个序号
2. 序号转为 62 进制 4 位字符串作为资源 ID
3. 创建数据库账号时同理，独立查找、独立自增
4. 资源 ID 一旦分配永久固定，不可变更

### 2.3 62 进制映射

内部使用整数自增，展示和传输使用 62 进制字符串：

```
0 → 0 | 10 → a | 36 → A | 62 → 10
1 → 1 | 11 → b | 37 → B | ...
...    | ...     | ...     |
9 → 9 | 35 → z | 61 → Z |
```

## 三、会话ID 生成规则

### 3.1 生成方式

5 位 62 进制（62⁵ ≈ 9.16 亿），按用户独立自增。

### 3.2 两类会话

| 类型 | 生成时机 | 有效期 | 说明 |
|---|---|---|---|
| 永久会话 | 用户注册时自动分配 | 永久 | 用户自身的身份标识 |
| 临时会话 | 管理员授权时创建 | 有期限 | 给其他用户的临时访问授权 |

### 3.3 会话属性

| 字段 | 说明 |
|---|---|
| session_id | 5 位 62 进制字符串 |
| 绑定用户 | 关联到堡垒机用户 |
| 类型 | permanent / temporary |
| 状态 | active / disabled / expired |
| 有效期 | expires_at，临时必填，永久为空 |
| 创建时间 | created_at |

### 3.4 会话状态流转

```
active ──→ disabled（管理员手动关闭）
  │
  └──→ expired（到期自动失效）

disabled ──→ active（管理员手动启用）
```

### 3.5 完整用户名唯一性

完整用户名 = `资源前缀 + 资源ID + 会话ID`。不同用户的会话ID独立自增，可能重复（如两个用户都有 session `00001`），但因为完整用户名中包含不同的资源 ID 和前缀，整体始终保持唯一。

## 四、解析与连接流程

### 4.1 用户名解析

```
用户输入: HaB12cD345
         │ │     │
         │ │     └── 会话ID (5位): cD345 → 查到用户，验证状态和有效期
         │ └──────── 资源ID (4位): aB12   → 查到目标账号，获取连接信息
         └────────── 资源前缀 (1位): H      → 判断资源类型
```

### 4.2 连接验证流程

1. **解析前缀**（第1位）→ 确定资源类型
2. **解析资源ID**（第2-5位）→ 查表获取目标连接信息
3. **解析会话ID**（第6-10位）→ 查表获取用户身份
4. **检查会话状态** → status 必须为 active
5. **检查会话有效期** → expires_at 未过期
6. **RBAC 权限校验** → 用户是否有该资源的连接权限
7. **全部通过** → 建立连接

### 4.3 SSH 连接

- 当前格式（废弃）：`admin+<32位hex>@127.0.0.1`
- 新格式：`ssh HaB12cD345@127.0.0.1 -p 47102`
- 解析入口：`internal/access/static.go` 的 `ParseLoginName()`，改为解析 10 位用户名

### 4.4 数据库连接

- 当前格式（废弃）：`db-<12位hex>`
- 新格式：`psql -h <host> -p <port> -U DaB12cD345`
- 解析入口：`internal/server/dbproxy/server.go`，改为解析 10 位用户名

## 五、数据模型

### 5.1 资源表扩展（HostAccount / DatabaseAccount）

| 字段 | 类型 | 说明 |
|---|---|---|
| resource_seq | int | 自增序号，用于生成资源 ID |
| resource_id | string(4) | 4 位 62 进制资源 ID |

### 5.2 会话表（新增或扩展）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 主键 |
| session_id | string(5) | 5 位 62 进制会话 ID |
| session_seq | int | 用户维度的自增序号 |
| user_id | string | 关联堡垒机用户 |
| type | string | permanent / temporary |
| status | string | active / disabled / expired |
| expires_at | timestamp | 过期时间（临时必填，永久为空） |
| created_at | timestamp | 创建时间 |
| created_by | string | 创建者 |

## 六、RBAC 权限模型

### 6.1 权限判断链路

```
Session → User → Roles → Permissions → Resource (host_account / database_account)
```

1. Session 识别用户（是谁）
2. RBAC 判断该用户对该资源的访问权限（能不能连）
3. 有效期由 Session 的 status 和 expires_at 控制

### 6.2 资源 ID 格式

RBAC 中资源标识使用 10 位完整用户名中的资源部分：

| 资源类型 | RBAC 资源标识 |
|---|---|
| host_account | `host_account:H<资源ID>` |
| database_account | `database_account:D<资源ID>` |
